The [`Presenter`](/big-break/docs/reference/types/interfaces/presenter/) is your
genre's UI-facing voice. The engine produces a neutral `runSummary`; the
presenter turns it into the ending screen, the act interstitials, and the trophy
room. Without one, a run reaches a generic finale; with one, it reaches *your*
game's finale.

<Aside type="note" title="Optional, piece by piece">
Every field here is à la carte: provide it and the UI uses it; omit it and the
UI falls back to a neutral default. You can ship with only `endings` filled in
and add the flavor generators later.
</Aside>

## The shape

```ts twoslash
interface RunState { path: string | null; seed: number | null; [k: string]: any; }
// ---cut---
interface Presenter {
  endings: Record<string, any>;         // path id + fail-state key → copy
  exitInterviews?: Record<string, any>; // fail-state branching epilogues
  wallItems?: any[];
  trophies?: any[];
  epilogue?: (state: RunState) => string;
  headlines?: (state: RunState, seed: number) => { text: string; src: string }[];
  dms?: (state: RunState, seed: number) => { from: string; text: string }[];
  discography?: (state: RunState) => any[];
}
```

`endings` is the one field you'll always fill: a map from each summit id (and the
engine's fail-state keys — `burnout`, plus any your manifest declares) to
`success` / `partial` / `failure` copy. Everything else is optional flavor.

## Endings, keyed by summit and fail state

The `endings` map has one entry per summit id, plus one per fail-state key. Each
entry is a small object of `success` / `partial` / `failure` copy the ending
screen renders. A pack reskins the engine's fail states here too — the label a
`burnout` or `debt` ending shows is just copy:

```ts
export const endings = {
  // one entry per summit id from the manifest…
  megastar: {
    success: { title: 'Megastar', body: 'Stadiums chant a name your mother didn’t give you.' },
    partial: { title: 'Almost', body: 'The arena tour becomes a theatre tour. Still a tour.' },
    failure: { title: 'The Long Way', body: 'Fame is a door that opened a crack, then a draft closed it.' },
  },
  // …plus one per fail-state key (burnout is universal; the rest are your manifest's):
  burnout:   { title: 'Burned Out', body: 'You gave it everything, and everything took it.' },
  cancelled: { title: 'Cancelled',  body: 'The scene moved on without an RSVP.' },
  debt:      { title: 'In the Red', body: 'The math finally caught up with the dream.' },
};
```

## Flavor generators are pure functions of state

`epilogue`, `headlines`, and `dms` are **pure** `state → copy` functions, seeded
off the run so they're reproducible. Because they read the run's actual state, an
ending describes what really happened — an epilogue can branch on the committed
path, a subsystem counter, or the money left on the table:

```ts
import { mulberry32 } from '../engine.js';

export function buildEpilogue(state) {
  const rng = mulberry32(state.seed ?? 0); // seed off the run, never the clock
  const lines = [];
  if (state.path === 'megastar' && state.fame > 100) {
    lines.push('The comeback documentary writes itself.');
  }
  if ((state.money ?? 0) < 0) lines.push('You left money on the table. On purpose, you tell people.');
  return lines[Math.floor(rng() * lines.length)] ?? 'The story keeps going after the credits.';
}
```

<Aside type="caution" title="Seed off the run, not the clock">
These generators must be deterministic — seed the RNG from `state.seed` (with
`mulberry32`), never from wall-clock time. A run's ending screen should be
identical on replay — and the automated browser playthrough exercises exactly
this path, so a crash here blocks publishing.
</Aside>

## Trophies

`trophies` are checked against the `runSummary`. Each has a `check(summary) =>
boolean` predicate, so awards describe real feats — winning a specific summit,
reaching the finale under an attrition threshold, hitting a partial ending. This
is how the trophy room feels like *your* game rather than a generic checklist.

## The division of labor

The engine decides *what happened* (the summary); the presenter decides *how it's
told*. Keep game logic out of the presenter — it should read state and return
copy, nothing more. That separation is what lets the same UI shell render any
genre.
