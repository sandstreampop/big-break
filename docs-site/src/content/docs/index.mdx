## What you're building on

Big Break is a **3-act swipe-roguelike engine**. The core runs one lifecycle —
deck assembly → draw a card → swipe left or right → roll to a `bad` / `good` /
`incredible` tier → apply effects → act breaks → crossroads → finale — over a
seeded, resume-safe RNG. It knows nothing about *your* game.

A game is a **`Pack`**: a manifest, a deck, optional subsystems, and a
presenter. The whole surface is typed, and a truly unknown key is a compile
error — so the tier of a swipe outcome is exactly one of three things, and
nothing else:

```ts twoslash
type Tier = 'bad' | 'good' | 'incredible';
const rolled: Tier = 'good';
//    ^?
```
