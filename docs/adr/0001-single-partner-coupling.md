# Single-partner coupling with temptation, not a connection portfolio

The Coupling subsystem models the Islander as coupled with exactly **one**
Partner at a time, tracked by a single Bond value; the "juggling" flavour is
delivered as Temptation *cards* that test that couple, not as a second ledger.

We considered a **connection portfolio** — several simultaneous connections each
with its own Bond — which would give more strategic (Savvy) expression. We
rejected it because a single-partner model keeps Bond one clean number the
Summit gate for *The Real Thing* can read directly, keeps run state light, and
sharpens the Loyalty-vs-Savvy tension the game is built on rather than blurring
the loyalty fantasy behind a spread. It is also the structure the show actually
uses.

This is hard to reverse: the relationship model shapes the run-state schema and
the whole deck's framing, so switching to a portfolio later would be a large
rework.
